Speicherorte

Anwendungsdaten

Die Anwendung speichert alle lokalen Einstellungen standardmäßig in Windows 10/11 unter:

C:\Benutzer\<Benutzername>\AppData\Local\BikeDB<version>

Bilddateien

Das Verzeichnis von Bilddateien („Bilderordner“) kann unter „Einstellungen“ frei gewählt werden. Es sollte ein dedizierter Ordner sein, der nur für Bilder innerhalb der BikeDB verwendet wird. Dieser Ordner könnte z.B. im Userverzeichnis \Bilder oder \Dokumente angelegt werden. Empfohlen wird zudem, dass der Ordner Teil regelmäßiger Backups ist.

Bilddateien werden von BikeDB z.B. für Strecken, Höhenprofile oder Fahrzeuge verwendet.

Google Earth

Ich benutze selbst die Desktop-Version von Google Earth, um damit Bilddateien und Strecken zu erstellen. Dies geschieht mit Pfad erstellen, dem Snipping Tool von Windows und einem Bildbearbeitungsprogramm, z.B. Paint.NET. Die so erstellten Dateien für Höhenprofile und Strecken werden dann im Bilderordner gespeichert und stehen so in BikeDB zur Verfügung.

Damit Google Earth direkt aus BikeDB heraus verwendet werden kann, muss unter „Einstellungen“ der Pfad zur EXE von Google Earth eingetragen werden. Bei der Standardinstallation findet man diesen unter Windows hier:

C:\Program Files\Google\Google Earth Pro\client

Ein falscher Pfad führt zu einer Fehlermeldung.
