Multi-User

Die vorliegende Version von BikeDB hat eine eigene Benutzerverwaltung, die unabhängig von vorhandenen Windows- oder Netzwerknutzern ist. Dies bedeutet, dass alle Nutzer in BikeDB neu angelegt werden müssen. Es bedeutet aber auch, dass nicht jeder Nutzer eines Computers automatisch Zugriff auf die Anwendung hat.

Rollen

Es gibt zwei Arten von Anwendern innerhalb von BikeDB: normale Anwender und Administratoren. Nur letztere haben Zugriff auf alle Daten und können neue Anwender anlegen, normale Anwender können nur ihre eigenen Daten bearbeiten.

Ein neuer Benutzer muss daher einen Administrator bitten, einen neuen Account anzulegen.

Standard Administrator

Der erste Datenbank-Benutzer (ID 0) ist ein Administrator, den man nicht löschen kann. Alle Einträge, die im Auslieferungszustand von BikeDB vorhanden sind (z.B. „Deutschland“ als erstes Land), sind diesem Administrator zugeordnet. Dies bedeutet auch, dass der Standard Administrator alles anlegen kann, was allen Anwendern zur Verfügung stehen soll, z.B. Länder, Städte oder Fahrzeughersteller., sodass diese nicht individuell von jedem Benutzer angelegt werden müssen.

Der Standard Administrator wird auch benötigt, um den allerersten Benutzer anzulegen oder die Anwendung auf eine Einzelplatz-Benutzung umzustellen. Hierbei wird der Standard Administrator zum einzigen Anwender, aber es gibt keinen Login oder Logout mehr, sodass man sich gut überlegen sollte, ob man diesen Modus verwenden will. Ansonsten sollte man aus Sicherheitsgründen immer mindestens das Passwort des Standard Administrators ändern, um unbefugtem Zugriff vorzubeugen.

Username: admin
Passwort: admin
